Standards

Writing a specification that builds on the Web Platform requires in-depth knowledge of its core technologies: HTML5, Web IDL, and ECMAScript. I'm able to edit and review specifications to make sure they will work well with the core Web Platform.

Prototypes

Independent verification of a specification through prototyping can be a cost effective way of showing the technical completeness of a specification and demonstrating that a specification works as intended. I'm able to prototype specifications in browser environments, Node.js, and Java.

Quality Assurance

Test suites are a key component to achieving interoperability in the standardization process. Knowing how much and what areas to test can improve the interoperability of products. I'm able to design, code, or review test suites as well as produce high quality implementation reports.
